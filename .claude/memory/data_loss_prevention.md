description: 绝不能在初始化/恢复流程中自动删除用户数据，清理逻辑必须在操作实际完成后以保守方式执行
type: feedback
---

绝不能在初始化/恢复流程中自动删除用户数据。

**Why:** `refreshAll` 中对所有过去提醒执行 `updateTask(id, {'reminderAt': null})`，一次 APP 重启清除了 42/45 个任务的提醒时间。"过去时间"不等于"已触发"——APP 关闭期间错过的提醒仍是有价值的用户数据。

**How to apply:**
1. `refreshAll` / 启动恢复流程的语义是"恢复"，不是"清理"
2. 清理逻辑必须在提醒实际弹出后执行（如 Windows Timer 回调中），且只清除已触发的那一条
3. 任何自动删除用户数据的代码都需要宽限期和用户可感知的反馈
4. 修改涉及数据删除的代码前，先问：如果这里有 bug，最坏会丢什么？
5. 数据库中的过期值宁可保留（死数据），也不要主动删除（数据丢失）
